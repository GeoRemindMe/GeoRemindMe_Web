geouser.views.confirm(request, user, code)¶

Descripción: confirma el email de usuario

Parámetros:
  • user (string) – nombre de usuario
  • code (string) – código de confirmación
Devuelve:

En caso de que todo vaya correctamente solicitar identificarse al usuario. En caso contrario devuelve un mensaje de error
geouser.views.get_followers(request, userid=None, username=None, page=1, query_id=None)¶

Descripción: Obtiene la lista de followers de un usuario, si no se recibe userid o username, se obtiene la lista del usuario logueado.

Parámetros:
  • userid (string) – id del usuario (user.id)
  • username (string) – nombre del usuario (user.username)
  • page (int) – numero de pagina a mostrar
  • query_id (int) – identificador de busqueda
Devuelve:

lista de tuplas de la forma (id, username), None si el usuario tiene privacidad

geouser.views.login(request)¶

Descripción: Recibe petición de autenticación vía POST y si los datos son correctos se llama a la función login_func que inicializa, crea la sesión y devuelve un mensaje en caso de éxito.

Devuelve:En caso de recibir el POST devuelve un string con posibles errores y redirect con la URL a la que habría que dirigir. En caso contrario renderiza la plantilla de login.
geouser.views.login_facebook(request)¶

Descripción: Utiliza el protocolo OAuth para solicitar al usuario que confirme que podemos usar sus sesión de Facebook para identificarlo. Así si el usuario está identificado en Facebook no necesitará rellenar el formulario de login de GeoRemindMe. login_twitter()

Devuelve:En caso de exito redirige al panel y en caso contrario redirige al panel de login.
geouser.views.login_google(request)¶
Descripción: Comprueba si ya te has dado de alta en la App con tu cuenta de Google
para entrar automáticamente y sino pedirte permiso para hacerlo. Además en caso de éxito inicializa la sesión para dicho usuario.
Devuelve:En caso de exito llama a geouser.funcs.login_func() y redirige al panel. En caso contrario renderiza la plantilla de login.
geouser.views.login_twitter(request)¶

Descripción: Utiliza el protocolo OAuth para solicitar al usuario que confirme que podemos usar sus sesión de Twitter para identificarlo.

Así si el usuario está identificado en Twitter no necesitará rellenar el formulario de login de GeoRemindMe.

Devuelve:En caso de exito redirige al panel y en caso contrario redirige al panel de login.
geouser.views.logout(request)¶

Descripción: Elimina la sesión de usuario y redirige al usuario.

Devuelve:Redirige al usuario a donde le diga la función login_panel.

geouser.views.register(request)¶

Descripción: Recibe una petición de registro vía POST e intenta crear un usuario; en caso de éxito le enviará un email para confirmar su cuenta.

Parámetros:request (array :return: En caso de éxito un google.appengine.api.user y un formulario de registro. En caso de que no se reciba POST redirige al panel de registro) – array con un email y dos contraseñas
